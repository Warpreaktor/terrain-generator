# Архитектура terrain-generator

## Модули

### `terrain-core`
Чистая библиотека для работы с сеткой высот.

Содержит:

- модель сетки;
- абстракцию хранения высот;
- низкоуровневые операции над рельефом;
- шумы;
- простую статистику.

### `terrain-lab`
Отдельное desktop-приложение для визуальной проверки результата.

Содержит:

- окно на libGDX;
- top-down просмотр;
- переключение палитры;
- отображение сетки;
- выбор клетки;
- просмотр финальной карты и промежуточных шагов pipeline.

## Главные правила

1. `terrain-core` не зависит от libGDX.
2. `terrain-core` ничего не знает про предметный смысл мира.
3. Любая высокая логика должна жить вне `terrain-core`.
4. `terrain-lab` — это потребитель библиотеки, а не вторая реализация тех же алгоритмов.

## Почему так

Эта схема позволяет отдельно:

- развивать математику рельефа;
- тестировать операции без графики;
- визуально смотреть результат через внешний стенд;
- позже заменить хранение на чанковое без ломки публичного API.

## Что можно наращивать дальше

- маски применения операций;
- отдельные фильтры террасирования и разломов;
- экспорт heightmap;
- сериализацию сетки;
- внутреннее чанковое хранение;
- высокий инструмент композиции ландшафтов поверх `terrain-core`.

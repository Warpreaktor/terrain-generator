# terrain-generator

Проект для построения и исследования рельефной сетки.

Сейчас в проекте два модуля:

- `terrain-core` — чистая библиотека работы с сеткой высот и низкоуровневыми операциями над рельефом.
- `terrain-lab` — отдельное desktop-приложение на libGDX для визуальной проверки результата в виде top-down сетки.

## Архитектурная идея

`terrain-core` ничего не знает про биомы, геотермию, воду, жизнь и прочую предметную магию.

Он умеет только:

- хранить сетку высот;
- изменять её через низкоуровневые операции;
- считать простую статистику;
- выполнять последовательность операций через pipeline.

`terrain-lab` использует `terrain-core` как внешний клиент и просто показывает результат глазами.

## Структура проекта

```text
terrain-generator/
├── terrain-core/
└── terrain-lab/
```

## Требования

- JDK 21
- Gradle 8.14.3 или совместимый

Gradle официально поддерживает запуск на JVM 17–26, а Java 21 полноценно поддерживается современными версиями Gradle.  
Для macOS и LWJGL3 в документации libGDX всё ещё рекомендуется запускать приложение с `-XstartOnFirstThread`, поэтому этот флаг добавлен в задачу `terrain-lab:run` автоматически. citeturn292232search0turn292232search2turn292232search11

## Запуск

### Если у тебя установлен Gradle
```bash
gradle :terrain-lab:run
```

### Если Gradle не установлен
В проекте есть `./gradlew`. Это bootstrap-скрипт: он скачает Gradle `8.14.3` в локальную папку проекта и дальше запустит сборку.

```bash
./gradlew :terrain-lab:run
```

## Что умеет `terrain-lab`

- показывает сетку высот сверху;
- умеет переключать цветовую схему;
- умеет включать и выключать видимую сетку;
- даёт регенерацию карты по новому `seed`;
- показывает информацию по клетке под курсором.

## Управление в `terrain-lab`

- `R` — сгенерировать новую карту с новым seed
- `C` — переключить цветовую схему
- `G` — включить или выключить сетку
- `1` — показать только результат финальной карты
- `2` — показать карту после каждого шага pipeline по кругу
- `W`, `A`, `S`, `D` — двигать камеру
- `Q`, `E` — zoom out / zoom in
- левая кнопка мыши — выбрать клетку

## Дальнейшее развитие

Следующий слой должен жить **поверх** этой библиотеки и использовать её как инструмент:

- генератор типовых форм рельефа;
- более высокий редактор или конструктор ландшафтов;
- раздельные математические маски;
- чанковое хранение как внутренняя оптимизация;
- экспорт heightmap и последующая интерпретация поверхности.

Сейчас проект сознательно ограничен: он строит поверхность, а не объясняет миру, чем эта поверхность является.
